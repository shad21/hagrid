Grille

La première chose à faire est de modifier, au besoin, quelques propriétés dans le fichier _grid.scss
A savoir: la largeur maximale du container principal, le nombre de colonnes souhaitées, la largeur des gouttières, les media queries pour tablette et bureau. Par défaut, les valeurs sont les suivantes: 

$wrapper-max-width:1200px !default; /* Définir en px, %, rem ou em */
$grid-columns:12 !default;
$gutter:0% !default; /* Définir en % uniquement */
$tablet:40.063em !default; /* 641px - Définir en px, em ou rem */
$desktop:64.063em !default; /* 1025px - Définir en px, em ou rem */

Pour créer une nouvelle page il faut tout d'abord définir un élément <div> avec la classe .wrapper. C'est le conteneur principal. Il englobera la page web. Il faut ensuite déclarer un élément <div> en lui associant la classe .row.

Syntaxe

Il faut spécifier la largeur de chaque colonnes en utilisant les classes sm-#, md-# et lg-#.
Si l'on souhaite avoir 2, 4 et 6 colonnes dans une row pour tous types d'écrans (mobile, tablette, bureau), il suffit de déclarer les classes .sm-2, .sm-4 et .sm-6.

Hide-and-seek

Il est possible de modifier le nombre de colonnes selon le type de média sur lequel on est. C'est là le but 1er du framework. Pour ce faire, il faut jouer avec les classes .sm-show, .hide-only-sm , .md-show, .md-hide, .lg-show et .lg-hide.

Exemples

Pour bien comprendre, visualiser le 1er exemple en partant du format smartphone puis, étendre jusqu'au format tablette et enfin bureau.

 Je suis visible uniquement sur tablette. .md-show
 Je suis visible uniquement sur bureau. .lg-show Je serai invisible à partir du format tablette. .md-hide
